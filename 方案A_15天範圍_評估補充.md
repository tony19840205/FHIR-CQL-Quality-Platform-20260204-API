## 📊 FHIR系統的時間邏輯

### FHIR資料的時間戳記

FHIR系統有兩種時間概念：

#### 1️⃣ 資料內容的時間（Resource內的日期）
```json
{
  "resourceType": "Encounter",
  "period": {
    "start": "2026-01-01T08:00:00+08:00",  // 就診時間
    "end": "2026-01-03T10:00:00+08:00"
  }
}
```
**這是臨床事件發生的時間**

#### 2️⃣ 資料上傳的時間（Meta.lastUpdated）
```json
{
  "resourceType": "Encounter",
  "meta": {
    "lastUpdated": "2025-12-24T15:30:00+08:00"  // 上傳時間
  },
  "period": {
    "start": "2026-01-01T08:00:00+08:00"
  }
}
```
**這是資料被上傳/修改的時間**

---

### 系統查詢的運作方式

**CQL查詢會看「資料內容的時間」，不是「上傳時間」**

#### 範例：查詢2026 Q1的住院資料

```cql
define "Q1_2026_Encounters":
  [Encounter] E
    where E.period.start during Interval[@2026-01-01, @2026-03-31]
```

**查詢邏輯**:
- ✅ 只看 `period.start` (就診開始時間)
- ❌ 不看 `meta.lastUpdated` (上傳時間)
