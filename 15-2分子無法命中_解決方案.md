# 15-2指標分子無法命中問題 - 解決方案

## 問題診斷

根據調試記錄 [INDICATOR_15-2_DEBUG_LOG.md](INDICATOR_15-2_DEBUG_LOG.md)：
- **分母**: 6個患者 ✓ 正確找到
- **分子**: 0個感染 ✗ 應該要有2個

## 根本原因

**Patient資源缺少`identifier`欄位**

### CQL邏輯分析

根據 [Indicator_15_2_Total_Knee_Arthroplasty_90Day_Deep_Infection_3249.cql](cql/Indicator_15_2_Total_Knee_Arthroplasty_90Day_Deep_Infection_3249.cql)：

```cql
define "TKA Procedures":
  [Procedure] P
    where exists (P.code.coding C where C in {
        "64164B", "97805K", "97806A", "97807B", "64169B",
        "609588000", "179344006", "265157000"
      })
    and P.status ~ 'completed'

define "Deep Infection After TKA":
  [Procedure] PI
    where exists (PI.code.coding C where C in {"64053B", "64198B"})
    and PI.status ~ 'completed'
    and exists (
      "TKA Procedures" TKA
        where TKA.subject.reference = PI.subject.reference  // 關鍵匹配邏輯
        and (PI.performed as FHIR.dateTime).value in Interval[
          (TKA.performed as FHIR.dateTime).value,
          (TKA.performed as FHIR.dateTime).value + 90 days
        ]
    )
```

**關鍵發現**：
- CQL透過`TKA.subject.reference = PI.subject.reference`來匹配患者
- 這需要TKA手術和感染手術的`subject.reference`完全一致
- 但是，如果Patient沒有identifier，CQL在搜尋時可能無法正確建立患者索引

### 原始資料問題

**舊的generate_15_2_tw10001_06.py (第23行)**：
```python

bundle["entry"].append({

        "resourceType": "Patient",
```

## 解決方案

### 修正後的Patient資源

```json
{
  "resourceType": "Patient",
  "id": "TW10001",
  "identifier": [
    {
      "system": "http://www.cgmh.org.tw/patients",
      "value": "TW10001"
    }
  ],
  "gender": "male",
  "birthDate": "1955-03-15"
}
```

### 關鍵修改點

1. **Patient.id統一使用TW10001格式**（不是total-knee-patient-001）
2. **新增identifier欄位**，value與id一致
3. **所有reference都使用Patient/TW10001格式**

## 技術細節

### 為什麼identifier很重要？

雖然CQL的匹配邏輯看起來只用`subject.reference`，但：

1. **FHIR伺服器索引**：許多FHIR實作會為identifier建立專門的索引
2. **跨資源查詢**：identifier是FHIR推薦的患者識別方式
3. **測試資料一致性**：與2025Q4原始資料保持一致（TW10019也有identifier）

### CQL執行流程

1. **搜尋TKA Procedures** → 找到6個（透過code=64164B等）
2. **搜尋感染Procedures** → 找到2個（透過code=64053B）
3. **匹配患者** → 比對`subject.reference`
4. **檢查時間** → 驗證感染在TKA後90天內
5. **計算比率** → 2/6 = 33.33%
